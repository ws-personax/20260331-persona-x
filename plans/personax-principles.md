# PersonaX Principles
## The Constitution of PersonaX

## 1. 존재 이유
PersonaX는 AI 채팅앱이 아니다.

PersonaX는 사람이 더 좋은 결정을 내리도록 돕는 Decision Operating System이다.

목표는 답변 생성이 아니라 의사결정의 품질 향상이다.

## 2. 핵심 철학
- 사람보다 AI가 주인공이 되지 않는다.
- Persona는 답을 주는 존재가 아니라 Decision Participant이다.
- 정답보다 기준을 만드는 것이 중요하다.
- 많이 기억하는 AI보다 제대로 기억하는 AI가 더 중요하다.

## 3. 아키텍처 원칙
### ① Room First
Room이 제품의 중심이다.

### ② Speaker Before Persona
누가 말했는지가 어떤 Persona가 답하는지보다 먼저다.

### ③ Decision Before Memory
결정이 먼저이고 Memory는 Decision에서 파생된다.

### ④ Memory Separation
Room Memory와 Personal Memory는 절대 섞지 않는다.

### ⑤ Context Boundary
Context Bleeding은 가장 위험한 버그이다.

## 4. Persona 원칙
### JACK
- 책임
- 선택
- 실행

### LUCIA
- 감정
- 회복
- 공감

### RAY
- 정의
- 분해
- 검증

### ECHO
- 패턴
- 원칙
- 구조

각 Persona는 다른 Persona 흉내를 내지 않는다.

## 5. 제품 원칙
- 기능보다 품질
- SNS보다 답변 품질
- Memory보다 Speaker
- 다운로드보다 재방문
- 칭찬보다 행동

## 6. 개발 원칙
- 작은 PR
- 회귀 방지
- 문서 먼저
- 구현 나중
- Single Source of Truth
- Prompt보다 Architecture

## 7. 성공 기준
사용자가 더 오래 쓰는가.

사용자가 더 자주 돌아오는가.

사용자가 더 좋은 결정을 하는가.

이 세 가지가 다운로드 수보다 중요하다.

## 8. 하지 않을 것
- 무분별한 Memory
- Context Bleeding
- Persona 혼합
- SNS 기능 남발
- 과도한 기능 추가
- 복잡한 UI

## 9. 장기 비전
PersonaX는 다음 순서로 성장한다.

Decision OS
↓
Decision Rooms
↓
Decision Network
↓
Decision Platform

## 10. 마지막 선언
"PersonaX는 AI를 만드는 프로젝트가 아니라,
사람이 더 좋은 결정을 내릴 수 있는 환경을 만드는 프로젝트이다."
