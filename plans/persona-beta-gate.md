# Plan: PersonaX Beta Gate Criteria

## 1. 베타 검증 목적
PersonaX가 단순히 흥미로운 아이디어인지, 아니면 반복 사용될 수 있는 제품인지 판단하기 위한 객관적 기준을 정의한다.

검증 항목:
- 답변 품질 검증
- 반복 사용 가능성 검증
- Room / Speaker 필요성 검증
- Memory 가치 검증

## 2. 베타 대상
- 지인 5~10명
- 창업 / 투자 / 관계 / 가족 의사결정이 있는 사람
- 2~3주 사용 기준

## 3. 핵심 질문
- 사용자가 다시 들어오는가
- 같은 주제를 다시 꺼내는가
- Persona를 직접 호출하는가
- Decision Summary를 다시 보는가
- Review를 의미 있게 느끼는가
- Room이 있으면 더 쉽게 쓰는가

## 4. 측정 지표
- 1주 재방문율
- 2번째 고민 입력률
- Persona 2회 이상 호출률
- 같은 Room 재방문 횟수
- Decision Summary 열람률
- Review Card 재진입률
- 지인 초대 유지율

## 5. 통과 기준
- 5명 중 3명 이상이 2회 이상 사용한다
- 5명 중 2명 이상이 "다시 써보고 싶다"는 반응을 보인다
- 1명 이상이 Room 초대 또는 공유 행동을 한다
- 최소 3개 이상의 실제 의사결정 기록이 생성된다

## 6. 실패 기준
- 첫 사용 후 재방문이 없다
- 답변은 좋지만 다시 쓸 이유가 없다는 반응이 나온다
- Room 없이도 충분하다는 반응이 반복된다
- Memory / Review가 불필요하다는 반응이 나온다
- Persona 차이를 느끼지 못한다

## 7. SNS / Room 검증 기준
- Room이 재방문을 높이는가
- 지인이 있으면 질문이 더 자연스러운가
- `@JACK` / `@RAY` 호출이 실제로 쓰이는가
- 혼자 쓰는 것보다 함께 쓰는 것이 더 편한가

## 8. Memory 검증 기준
- 사용자가 "지난번 결정"을 다시 보고 싶어 하는가
- Room Memory와 Personal Memory의 차이를 이해하는가
- Review가 부담이 아니라 도움이 되는가

## 9. 주의할 점
- 칭찬보다 재방문 행동을 본다
- "좋다"보다 "또 썼다"가 중요하다
- 기능 요청보다 반복 행동이 중요하다
- 베타 인원이 적어도 행동 로그는 의미가 있다

## 10. 다음 판단
- 통과하면 Room / Speaker 구현을 계속한다
- 애매하면 답변 품질을 추가 개선한다
- 실패하면 Memory보다 사용 동기를 재설계한다
