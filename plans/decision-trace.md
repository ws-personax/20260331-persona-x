# Decision Trace

## 1. Decision Trace 정의

Decision Trace는 PersonaX가 각 페르소나 답변에 대해 "왜 이런 판단을 했는가"를 설명하고 추적하기 위한 내부 기준 시스템이다.

일반 AI는 답변만 제공하지만, PersonaX는 RAY/JACK/LUCIA/ECHO가 서로 다른 판단 기준으로 같은 질문을 본다.

그 기준 차이가 충돌이고, 그 충돌을 눈에 보이게 만드는 것이 Decision Trace다.

## 2. Decision Trace의 목적

- 페르소나별 답변 근거를 설명 가능하게 만든다.
- 왜 4명이 다른 말을 하는지 사용자가 이해할 수 있게 한다.
- 답변 품질을 내부적으로 평가한다.
- 부족한 기준을 찾아 다음 답변 품질을 올린다.
- Decision Summary가 어느 기준을 중심으로 결론을 냈는지 설명할 수 있게 한다.

## 3. 페르소나별 기준과 체크리스트

### RAY = 정량 기준

체크리스트:

- 숫자/비율/기간/비용/리스크 중 하나 이상을 사용했는가?
- 비교 기준을 제시했는가?
- 검증 가능한 조건을 남겼는가?
- 추측을 사실처럼 말하지 않았는가?

### JACK = 행동 기준

체크리스트:

- 선택을 제시했는가?
- 포기할 것을 드러냈는가?
- 책임 또는 대가를 설명했는가?
- 다음 행동이 명확한가?
- 사용자를 공격하지 않고 회피/미루기/불명확한 선택을 공격했는가?

### LUCIA = 정성 기준

체크리스트:

- 감정의 무게를 다뤘는가?
- 관계와 상처를 고려했는가?
- 회복 가능성을 제시했는가?
- 판단을 회피하는 위로에 머물지 않았는가?

### ECHO = 패턴 기준

체크리스트:

- 반복을 포착했는가?
- 구조를 설명했는가?
- 장기 결과 또는 누적 효과를 제시했는가?
- 질문형이 아니라 판결형으로 닫았는가?

## 4. 신뢰도 계산 방식

초기 설계는 단순 점수 방식으로 한다.

각 Persona별 checklist 항목을 0/1로 평가한다.

충족 항목 수 / 전체 항목 수 * 100 = 기준 충족도.

예시:

RAY checklist 4개 중 3개 충족

→ RAY Trace Score = 75%

이 점수는 사용자에게 바로 노출하지 않을 수 있다. 초기에는 내부 QA와 디버깅 용도로 사용한다.

## 5. Criteria Conflict

PersonaX의 핵심 가치는 4명이 같은 답을 하는 것이 아니라, 서로 다른 기준으로 충돌하는 것이다.

예시:

경제적 자유 질문

- RAY: 정량 기준 88%
- JACK: 행동 기준 91%
- LUCIA: 정성 기준 94%
- ECHO: 패턴 기준 73%

이 경우 Summary는 다음처럼 설명할 수 있다.

> 정량 기준보다 정성 기준과 행동 기준에서 더 강한 신호가 나왔다.

## 6. 내부 QA 활용 방법

Decision Trace는 초기에는 사용자에게 직접 보여주지 않는다.

활용:

- Persona 답변 품질 점검
- RAY가 숫자만 말하고 검증을 빠뜨렸는지 확인
- JACK이 사용자를 공격했는지 확인
- ECHO가 물음표로 끝났는지 확인
- Summary가 어떤 기준을 반영했는지 확인

## 7. 향후 구현 순서

### Step 1

`plans/decision-trace.md` 문서 추가.

### Step 2

Decision Criteria가 `rules.ts`에 반영되었는지 확인.

### Step 3

Stage 3 프롬프트에 Self-Check 규칙을 추가.

단, 추가 LLM 호출 없이 같은 호출 안에서 Generate → Self-Check → Output 방식으로 처리한다.

### Step 4

내부 QA helper로 Trace Score 계산.

### Step 5

Decision Summary가 어떤 기준을 중심으로 결론을 냈는지 설명하도록 개선.

### Step 6

향후 사용자에게 선택적으로 "왜 이런 결론이 나왔는지"를 보여주는 UI 검토.

## 8. 금지 원칙

- Critique를 위해 별도 LLM 호출을 추가하지 않는다.
- Generate → Critique → Refine 구조는 비용 증가 때문에 현재 단계에서는 사용하지 않는다.
- PersonaX는 Generate → Self-Check → Output 구조를 우선한다.
- Decision Trace는 답변을 길게 만드는 장식이 아니다.
- Decision Trace는 판단 근거와 품질을 추적하는 내부 시스템이다.
- 사용자에게 보여줄 때도 점수 경쟁처럼 보이게 하지 않는다.
